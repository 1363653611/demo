 CGLIB 动态代理

    CGLIB 代理不依赖接口，是依赖类来实现的接口。
    原理：
        1.对指定的委托类，实现一个子类，并用子类来重写委托类(父类)中的业务方法。来实现代理。
        2.代理类对象是由Enhancer 类来创建的。
    CGLIB 动态代理的类的模式：
        1.查找目标类上所有非final 的public 的方法（final 不能被重写）。
        2.将这些方法的定义转换为字节码
        3.将组成的字节码转换为相应的代理的Class 对象，然后通过反射获得代理类的实例对象
        4.实现 MethodInterceptor 接口。用来处理代理类上所有方法的请求。


  操作步骤：
        1.引入CGLIB 所需的第三方jar包
        2.编写委托类
        3. 编写 MethodInterceptor 的实现类
